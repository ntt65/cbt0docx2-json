### 🏆 [CBT 데이터 변환 엔진 V12.8] 개발 상세 보고서
**주제:** Cell-Split Delimiter 아키텍처 및 하이브리드 파싱 엔진 도입

---

### 1. 업데이트 배경 (배경 및 문제점)

기존 엔진(V12.4 이전)은 문제 원본 워드 문서의 표(Table) 구조에서 지문/보기(Cell 0)와 해설/정답(Cell 1)을 단순 1차원 리스트로 선형화(Flatten)한 뒤, 보기 기호(①~④) 및 `정답` 키워드만을 델리미터 삼아 쪼개는 방식을 취하고 있었습니다.

하지만 새로운 워드 파일(`gas_CBT_2017_2025.docx`)이 유입되면서 다음과 같은 치명적 장애가 발생했습니다:
1. **가짜 문항 인식으로 인한 조기 쪼개짐 (Premature Split):** 해설 및 표 내부에 들어있는 단위 수식(예: `10[L] 이하`, `5분`, `24분` 등)이 문제 번호로 오인되어 블록이 중간에 난도질당했습니다.
2. **델리미터 소실 및 힌트 흡수:** 블록이 조기 단편화되면서 정답 키워드가 뒤로 밀렸고, 이에 따라 보기 4번(`④`) 추출 정규식이 멈출 곳(정답 델리미터)을 찾지 못해 남은 해설/힌트를 전부 보기 4번에 흡수하여 `hint` 필드가 공백이 되는 문제가 발생했습니다.

이 문제를 근본적으로 해결하기 위해 **물리적인 셀 분리자(Cell-Split Delimiter) 삽입 방식**과 **수식/단위 배제 정규식**을 결합한 하이브리드 엔진 V12.8을 설계 및 도입했습니다.

### 2. V12.8 핵심 패치 내용

#### ① 유니크 셀 구분자 (`|||CELL_SPLIT|||`) 도입
* **문서 선형화 단계 (`flatten_document`):** 
  스캔 도중 발견된 표가 문제를 담은 껍데기 표(Wrapper Table)일 경우, 좌측 셀(문제/지문/보기)과 우측 셀(해설/정답) 사이에 `|||CELL_SPLIT|||`이라는 확실하고 고유한 문자열 구분자를 강제로 끼워 넣습니다.
* **문항 파싱 단계 (`parse_question_block`):**
  문항 블록 내부에 `|||CELL_SPLIT|||` 구분자가 발견되면, 이를 기준으로 텍스트를 좌우로 선 분리(Physical Split)합니다.
  * **좌측 텍스트:** 오직 지문과 보기 ①~④만 포함하므로 보기 4번이 힌트를 침범할 여지가 원천 차단됩니다.
  * **우측 텍스트:** 오직 해설(힌트)과 정답만 포함하므로 정답 검출 후 남은 텍스트를 100% 온전하게 `hint` 필드로 추출합니다.

#### ② 텍스트 포맷 대응용 하이브리드 폴백(Fallback) 탑재
* 표가 아닌 일반 텍스트 형식으로 작성된 문항 영역의 경우, 구분자가 존재하지 않습니다.
* 구분자가 없을 경우 보기 4번(`④`) 추출 시 기존의 탐욕적(Greedy) 탐색을 배제하고, **줄바꿈(`\n`)이나 `정답` 기호를 만나면 매칭을 중단하는 전용 룩어헤드(`(?=\n|정답|$)`)를 탑재**하여 텍스트 모드에서도 보기와 힌트가 정확히 칼분할되도록 하이브리드 아키텍처를 구현했습니다.

#### ③ 초정밀 문제 번호 필터링 (`Q_PATTERN` 고도화)
* **수식 기호 배제 (`EXCLUDE_LIST`):** 숫자 뒤에 `[`, `=`, `:`, `/`, `+`, `-`, `*`, `×`, `[xX]`, `,`, 소수점 등의 수학 연산자와 기호가 오면 문제 번호 인식에서 즉시 배제합니다.
* **길이 및 대괄호 검증:** 번호 인식 패턴 통과 후에도 남은 텍스트가 대괄호(`[`)로 시작하거나, 전체 줄의 길이가 13글자 미만인 경우(예: `24분`) 가짜 문제 번호로 판정하고 패스합니다.

### 4. 최종 변환 성적표 (`gas_CBT_2017_2025.docx`)

* **총 변환 회차:** 18개 회차 (2017년 1회 ~ 2025년 3회)
* **총 추출 문항:** 1,080문항 (누락 0개, 중복 0개, 가짜 문제 0개)
* **회차별 무결성 성적:** 18개 회차 전 회차 **60문제 정원 완벽 보존 (100% 성공)**

#### 🎯 JSON 산출물 검증 데이터 (`data/gas/gas_questions.json`)
* **표 형식 문항 파싱 검증 (1번 문제):**
  ```json
  {
    "num": 1,
    "question": "다음 중 산소 없이 분해폭발을 일으키는 물질이 아닌 것은?",
    "options": [
      "아세틸렌",
      "히드라진",
      "산화에틸렌",
      "시안화수소"
    ],
    "hint": "중합폭발\n시안화수소 등 중합에 의해 폭발",
    "answer": 4
  }
  ```
  * 4번 보기가 `"시안화수소"`로 깔끔하게 쪼개지고, 해설 내용이 `hint` 필드로 완벽 분리되었습니다.
* **복잡한 해설 표 보존 검증 (2번 문제):**
  ```json
  {
    "num": 2,
    "question": "도시가스 사용시설인 배관의 내용적이 10[L] 초과 50[L] 이하일 때 기밀시험압력 유지시간은 얼마인가?",
    "options": [
      "5분 이상",
      "10분 이상",
      "24분 이상",
      "30분 이상"
    ],
    "hint": "내용적에 따른 기밀시험압력 유지시간\n내용적\n기밀시험 유지시간\n10[L] 이하\n5분\n10[L] 초과 50[L] 이하\n10분\n50[L] 초과\n24분",
    "answer": 2
  }
  ```
  * 해설 표의 모든 셀 텍스트가 줄바꿈 구분 형태로 깨지지 않고 고스란히 힌트에 저장되었습니다.
